Structured workflows for AI agents

Workflow Server is a Model Context Protocol (MCP) server that guides AI agents through structured, multi-step workflows. You describe a goal in natural language; the server gives the agent a defined process to follow — with checkpoints where you stay in control of the decisions that matter.

  • MCP compatible
  • TypeScript · Node.js 18+
  • MIT license

How it works

A single always-applied IDE rule bootstraps the agent. From there, the server handles workflow discovery, session management, and step-by-step navigation:

  1. Discover. The agent calls discover to learn the available workflows and the bootstrap procedure.
  2. Start a session. start_session returns a session token, and get_workflow returns the workflow structure, its workflow-level techniques and rules, and the first activity.
  3. Navigate. next_activity advances the session; get_activity returns the activity's full definition — steps, checkpoints, transitions — with its bundled techniques. get_resource lazy-loads reference material.
  4. Execute. The agent works through the activities, pausing at checkpoints for your decisions, with transitions governing the flow between activities.

MCP tools at a glance

The server registers 16 MCP tools across six concerns:

ConcernTools
Bootstrap (no session token)discover, list_workflows, health_check
Sessionstart_session, get_workflow_status, dispatch_child
Workflow / activity navigationget_workflow, next_activity, get_activity
Checkpoint flowyield_checkpoint, resume_checkpoint, present_checkpoint, respond_checkpoint
Techniques, resourcesget_technique, get_resource
Traceget_trace
