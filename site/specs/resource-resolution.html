Technique and resource resolution

Context windows are precious, so nothing loads that isn't needed. All behaviour is composed from techniques — markdown capability definitions referenced by ::-paths — resolved workflow-first with a shared meta fallback, delivered as bundles, and backed by reference material that loads only on demand.

Canonical source: docs/resource_resolution_model.md. The normative file anatomy is the technique protocol specification.

One kind of technique

A technique is a markdown file whose slug is its canonical id — standalone (techniques/{slug}.md) or grouped (techniques/{group}/TECHNIQUE.md with nested {sub}.md siblings; a nested technique is itself a technique). Its published shape: identity (id, version, capability), optional inputs and outputs (outputs may carry an artifact filename), an ordered protocol of imperative steps, and named rules — behavioural invariants that ride along with the technique.

References

Activities and workflows compose behaviour by listing ::-delimited references: [workflow::]technique[::nested…]. A same-workflow reference omits the workflow segment; a trailing segment that matches no nested technique is matched against the technique's rule names instead (a group prefix expands to every {group}-* rule). Anything that matches nothing surfaces in the unresolved bucket — never silently dropped.

Delivery at workflow and activity granularity

Agents never chain resolution calls at runtime. get_workflow returns the orchestrator bundle — the workflow's declared references plus the core orchestrator set the server auto-includes (engine traversal, checkpoint flow, dispatch, discipline). get_activity returns the worker bundle — the activity's references, the workflow's inherited techniques.activity, plus the core worker set (yield/resume, finalize, conduct rules). When a technique is delivered, ancestor containers' Initial/Final protocol blocks wrap the technique's own protocol recursively, and the combined sequence is renumbered.

Resources: lazy by design

Large reference material — CLI guides, templates, API notes — never rides along in bundles. A technique links a resource with an ordinary markdown hyperlink; at delivery the server rewrites those links into get_resource-callable refs ({id}[#section], or {workflow}/{id}[#section] across workflows). The agent fetches one only when it actually needs it, and a #section anchor narrows the fetch to a single heading — a template, not the whole file.
