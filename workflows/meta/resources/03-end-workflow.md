# End Workflow Reference

**Purpose:** Reference material for completing workflows. Provides templates and checklists. Flow and checkpoints are defined in the activity.

---

## Completion Summary Template

Use this template when presenting the final summary:

```markdown
## Workflow Complete

**Workflow:** [Workflow Name]
**Started:** [Start Date]
**Completed:** [Completion Date]

### Activities Completed
- [List of completed activities]

### Outcomes Achieved
- [Outcome 1]
- [Outcome 2]

### Key Decisions Made
| Checkpoint | Decision | Rationale |
|------------|----------|-----------|
| [Checkpoint] | [Option selected] | [Why] |

### Artifacts Created
- [List of artifacts/documents created]

### Follow-Up Items
- [Any deferred or future work items]

### Notes
[Any additional notes or observations]
```

---

## General Completion Checklist

Before ending any workflow, verify:

- [ ] All required activities are complete
- [ ] All blocking checkpoints have responses
- [ ] Workflow outcomes are satisfied
- [ ] Any external state is in expected condition
- [ ] Follow-up items are documented
- [ ] Final summary reviewed with user

---

## Completion Rules

1. **Never end with incomplete required activities**
2. **Ensure all blocking checkpoints have responses**
3. **Verify workflow outcomes are satisfied**
4. **Document any deferred or out-of-scope items**
5. **Present clear summary before finalizing**
