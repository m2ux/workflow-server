# Authoring Friction — Guard Ledger & Remediation History (Phase 1c)

Date: 2026-07-03. Every guard script exists because the authored corpus spans **two formats** — YAML definitions and markdown techniques whose I/O contracts are `###` headings — so no single-document schema can see the joins.

## 3. Recent signals

Repo root: guard-after-mass-fix rhythm dominates — check:anchors added *during* the subsumption wave (b05d5db2); lens lint during prism remediation (40f78f4d); guards made worktree-aware (cc2a1de4/a094cfd9) to gate authoring worktrees; binding-fidelity chasing the DSL's growing reference forms (e77c5b6a/4250105a); loop-body checkpoint instance-id fix (62829bb9) — authored-name vs runtime-name friction.

Workflows worktree: `35b35b86` "remediate 13 anti-pattern / binding findings (v3.16.0)" — mass-fix of recurring classes on a workflow already remediated once (**the errors regrow**); nine consecutive `subsume` commits (AP-85 wave); audience-bucket fixes (2778287e/802f492b) still occurring in June; guards accreting curated allowlists (`EXEMPT`, `PLACEHOLDER`, `PIPELINE_INTERNAL`) — itself a friction cost of enforcing semantics no schema carries.

**Overall shape:** two guards (check:steps, check:artifacts) police what the Zod schema could simply forbid. The rest police the seam the architecture created — contracts declared in markdown headings, consumed by YAML `{tokens}`, resolved by loader conventions only executable code can replay. The two biggest error classes (prose-in-structural-slots, duplicate declaration sites) were *invited* by the schema before being outlawed as anti-patterns and mechanized as guards.
