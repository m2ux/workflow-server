## 5. Compile pipeline

`score-compile` stages: **(1)** tsc type-check → **(2)** deterministic bundle → **(3)** sandboxed single-shot execution → **(4)** post-run determinism verification → **(5)** AST construction with source spans → **(6)** semantic passes (rule families, §7) → **(7)** lowering + canonicalization → **(8)** canonical IR bytes + content hash + diagnostics. The pipeline runs with no runtime present (Phase-1 shippable, R-PAR-4) and is exposable later as a governed `workflow_compile` MCP tool over the same entry point (R-GOV-4).

### 5.1 Deterministic sandbox (locked decision 1, operationalized)

The artifact's module graph is bundled deterministically and evaluated **exactly once** per compile (then once more for DET-004) in an isolated realm (SES / hardened `node:vm` class) with frozen intrinsics.

| Rule | Sev | Statement |
|---|---|---|
| **DET-001** | ERROR | Import allowlist: `@workflow-plane/score`, the workflow's generated `score.gen` module (types only — erased, no runtime body), and relative sibling definition modules of the **same workflow**. Nothing else — no node builtins, no third-party packages, no dynamic `import()`. Bundling order is deterministic (lexicographic module-path order); cross-activity imports are bounded to the workflow, and a node value reachable from two activities' definitions is ID-007 ERROR (ownership). |
| **DET-002** | ERROR | Poisoned ambient APIs: `Date`, `Math.random`, `crypto`, `performance`, timers, `process`, `fetch`, fs — replaced by throwing stubs; any touch aborts compilation with the offending call site. |
| **DET-003** | ERROR | Post-run structural walk: the exported definition value graph must be pure data — any surviving function, closure, promise, symbol-keyed property, or class instance is rejected. All callbacks (loop bodies) execute during the single build run; nothing executable survives into IR (R-P2/R-P3). |
| **DET-004** | ERROR | In-compile double run: the builder executes twice; the two canonical IR serializations are byte-compared. A mismatch aborts. (CI re-run hashing alone leaves a window where the first nondeterministic artifact ships locally.) |
| **DET-005** | ERROR | The default export must be a `WorkflowDef` or `ActivityDef`. |
| **DET-006** | ERROR | The builder threw: reported with the in-realm stack mapped through the bundle's source map. |

Insertion-order determinism: ECMA-262 enumerates string keys in insertion order **except** integer-like keys, which enumerate numerically. ID-004 (ERROR) therefore forbids numeric-like keys in `cases:`/`vars:`/branch/effect record literals — the one place authored order and enumeration order can diverge. (The legacy grammar's `Id` keys cannot collide with this rule, so the transpiled corpus is unaffected.) Checkpoint options and transitions are arrays and carry their order structurally.

### 5.2 Lowering and canonicalization

Lowering follows the R-IR-2/R-IR-3 table verbatim (§3.3's construct→node mapping). Additional canonicalization rules:

- **Field order** is fixed per production by the L2 grammar (§6); serialization is compact JSON (no insignificant whitespace), UTF-8, NFC-normalized strings, JCS-style escaping, integers in decimal without leading zeros.
- **Semantic order is authored order**: `run`/block items, branch arms and their sequences, checkpoint options, transitions, route branches, activity `inputs`, `activities[]`.
- **Non-semantic key sets sort lexicographically** by key: `vars`, `artifactLocations`, `Invoke.bindings`, `technique_args`, effect `setVariable` maps. (Serialized as pair-arrays, §6.2, so order is explicit in the bytes.)
- **Value reuse** lowers to one node + N id-references (the legacy declare-once/reference-many graph): nodes live in a per-activity node table; sequences hold `NodeId` strings.
- **Node-table order** is the depth-first first-occurrence walk of `run`, then declared `blocks` in authored order — deterministic from canonical structure, independent of builder internals.
- **Envelope**: every node is wrapped `NodeEnvelope { id, node, policy }` with the inert Phase-1 `ExecutionPolicy` (R-IR-ENV). The envelope is present in the bytes so Phase-2 policy attachment never reshapes the IR.
- **Hash**: sha256 over the canonical bytes is the signable artifact identity (R-GOV-4, R-X-1).

### 5.3 Stable-id discipline

- Authored ids pass **verbatim** into `NodeId`s (`<workflow>/<activity>/<kind>:<id>`), step/activity manifests, and trace events (R-PAR-3).
- Activity ordinals: `activities[]` position, or explicit `{ ordinal }` for corpus parity; ID-005 requires dense + unique. `artifactPrefix` is recomputed from the ordinal (`02`), never authored (ART-002).
- Anonymous items (messages, terminals, pass-throughs, anonymous sequences) get deterministic **structural ids** `<owner-id>#<index>` — identical for the TOON reader and Score by construction, stable under edits elsewhere, shifting only on local reorder, and absent from the trace-event vocabulary (legacy gives these no identity, so no parity loss).
- `outputOf()` refs lower to `NN.step.symbol` with `NN` derived at lowering — activity reorder re-derives instead of dangling. Raw qualified strings are checked by ID-006: existence (the ordinal names a preceding activity containing that step and output) plus an intent guard — under `--strict-refs` or after a detected reorder, a raw ref whose resolution *changed producer* since the last compile snapshot is an ERROR, closing the silent-misresolution window.

### 5.4 Source mapping

A small TS transformer injects `(file, line, col)` spans into constructor calls at bundle time — engine-independent, the only mechanism allowed to feed diagnostics in the canonical pipeline. Stack introspection is a diagnostics-only fallback (never canonical). Spans live in the AST sidecar (one of the three stored representations: Source, AST, IR — R-PIPE-2) and **never** in canonical IR: reformatting an artifact cannot change its IR hash.

### 5.5 Diagnostics model

One rule-id vocabulary across all stages, severity-graded (ERROR / WARN / INFO), source-spanned, emitted human-readable **and** as machine JSON (agents iterate on the JSON):

```json
{ "rule": "DEC-001", "severity": "WARN", "stage": "compile",
  "node": "work-package/requirements-elicitation/decision:platform-routing",
  "span": { "file": "activities/02-requirements-elicitation.ts", "line": 96, "col": 5 },
  "message": "match 'platform-routing' has no otherwise: branch; unmatched values pass through at runtime." }
```

- **tsc-stage** failures surface through the generated registry so they point at the offending literal (bad address ⇒ not-assignable-to-`keyof` with did-you-mean; bad bind key ⇒ excess-property naming the key); the GEN-002 fixtures pin both the firing and the text.
- **Legacy rule ids are preserved** wherever the rule survives (PROV/SYM→ID mapping noted in §7.3); severity grading survives hoisting (P4).
- **Profiles**: default (new authoring — MIG-001/ID-002 active) and `--parity` (corpus equivalence — MIG-001/ID-002 silent; EXPR-101 active, gating constructs with no TOON equivalent). Profiles change *severity routing only*, never semantics.

### 5.6 Compile passes (inventory)

In order: ID family (uniqueness, grammar, ordinals, numeric-key guard) → REF/RET resolution (byId, retry, rerun, goTo, transitions) → block reachability (FLOW-002) → loop containment (LOOP-002 sentinel walk, LOOP-101 tokens) → decision analyses (DEC-001 WARN, DEC-002 transitive closure) → TERM-101 reachability → scope-chain resolution (SCOPE-002, PROV-001, BIND-002/003/004, INT-001 tokens, ART-001 locations) → cross-activity resolution (PROV-002 residue, ID-006) → contract verification (ADDR re-verify, CTR-001 digest) → workflow shape (WF-001..003, TRN-001..003, RTE-001, CKPT-001/003) → GOV-001 → lowering → DET-004 double-run compare.
